# 為什麼要寫PCB？什麼時候應該更新？

## Answer

*定義：「專案管理書」（Project Control Book, PCB）是專案經理（PJM）使用的管理工具，用於整合與記錄任務的關鍵資訊，包含範疇、目標、時程、成本、重點工作、交付品質、委外資訊、專案風險等。執行過程中，專案經理（PJM）必須以PCB做為唯一依據跟成員溝通進度及變更，確保資訊一致、避免落差、並達成以下目的：
1.建立團隊對任務有共同理解，凝聚團隊共識。
2.明確各角色的任務與責任。
3.作為任務執行與變更的依據。
*使用情境：PCB應持續更新執行資訊，更新時機應依以下原則：

1.任務啟動
專案經理（PJM）須於 Kickoff 後7個工作天內，召集執行團隊（含提案經理（PLM））完成 PCB 初版，並建立對執行規劃的共識。
2.任務變更
只要發生以下情況，即需更新 PCB 並記錄變更內容：
 (1)任務、時程或資源調整。
 (2)客戶需求變更或契約調整。
 (3)人員異動（需附上「專案人員異動業務交接單」）
 (4)委外需求變動或委外契約調整。
 (5)達成重要里程碑（如交付、驗收等）。
3.定期檢視
建議於每階段或每季定期檢視，即使無重大變更，也能確保資訊正確性。
*提醒：
1.更新 PCB 時，應另存新檔，切勿覆蓋原檔，以確保版本可追溯。
2.須註明變更項目及原因，尤其是人員變更與任務調整。
3.PCB為內部管理文件，未經單位主管同意，不得對外提供（含客戶）。
